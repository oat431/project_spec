## Design Decisions Applied

| # | Decision | Implementation |
|---|----------|---------------|
| D1 | Flowero Guard IS Keycloak — no wrapper | Removed all references to "Spring Boot OAuth2 Resource Server." Guard = Keycloak container. Gate validates JWT locally against JWKS. |
| D2 | Nginx is the edge proxy | Added Nginx layer to all architecture diagrams. Gate is now internal on :8000 behind Nginx. |
| D3 | Domain scheme: `*.panomete.com` | All domain references changed from `panomete.local` to `*.panomete.com`. Path routing replaced with subdomain routing. |
| D4 | Foundation services get subdomains | Guard → `auth.panomete.com`. Discover → `discovery.panomete.com`. No longer routed through Gate. |
| D5 | Gate routes business APIs only | Gate scope reduced. `/auth/**` and `/eureka/**` routes removed from Gate config. Gate only routes `/api/{service}/**` patterns at `api.panomete.com`. |
| D6 | New port assignments | Guard: 8001. Discover: 8999 (BE) / 3999 (FE). Gate: 8000. Business: 8002–8007 (BE) / 3002–3007 (FE). |
| D7 | Shared infrastructure | Guard uses shared PostgreSQL 18 (no dedicated container). Gate rate limiting uses shared Valkey 9 (replaces in-memory). |
| D8 | Cloudflare handles TLS | Removed US-206 (TLS termination at Gate). All TLS-related ACs removed. |
| D9 | Observability deferred to Phase 2 | OBJ-05 timing updated. Uptime Kuma for basic monitoring in Phase 1. |

## Key Architecture Rules (Post-Update)

> These rules are now consistent across ALL documents:

1. **Nginx is the edge.** All external traffic hits Nginx first. Nginx routes by subdomain.
2. **Gate = business APIs only.** `api.panomete.com` → Gate :8000 → `/api/{service}/**`.
3. **Guard and Discover are direct.** `auth.panomete.com` → Guard :8001. `discovery.panomete.com` → Discover :3999/:8999. No Gate involvement.
4. **JWT validation is local.** Gate caches Keycloak JWKS on startup. Zero per-request calls to Guard.
5. **TLS is Cloudflare's job.** Internal traffic is plain HTTP on the trusted Docker network.
6. **Rate limiting is Valkey-backed.** Shared instance. Survives Gate restarts.
7. **Shared infrastructure.** PostgreSQL 18 (Guard + business services). Valkey 9 (Gate rate limiting). No dedicated DB containers.
8. **Observability is Phase 2.** Uptime Kuma for basic monitoring in Phase 1.
